# 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
ServerHUB - 远程服务器集群监控控制平面

### 1.2 产品定位
ServerHUB 是一款基于 Tauri 2 的跨平台桌面应用，为运维人员和开发者提供集中式的远程服务器集群监控能力。区别于传统的本地系统监控工具，ServerHUB 专注于多服务器的统一管理和实时监控。

### 1.3 产品愿景
成为运维人员首选的轻量级、高性能、易部署的服务器监控控制平面，让分布式服务器集群的健康状态一目了然。

---

## 2. 目标用户

### 2.1 主要用户画像

**运维工程师**
- 需要监控多台生产服务器
- 需要快速发现性能瓶颈和异常
- 需要历史数据回溯和趋势分析
- 需要告警通知机制

**开发者**
- 需要监控开发/测试环境服务器
- 需要了解应用运行时资源消耗
- 需要网络质量监控（延迟、丢包）

**小型团队/个人开发者**
- 预算有限，需要免费开源方案
- 不想依赖云服务，需要本地化部署
- 需要简单易用的界面

### 2.2 用户规模
- 初期目标：管理 5-50 台服务器的中小型团队
- 长期目标：支持 100+ 服务器的大规模集群

---

## 3. 核心功能

### 3.1 服务器管理

**功能描述**
- 添加/删除/编辑服务器配置
- 支持多种数据源适配器（node_exporter、Glances、Go Agent）
- 支持多种访问方式（私有网络、SSH 隧道、网关代理）
- 服务器分组和标签管理

**用户故事**
- 作为运维工程师，我希望能够快速添加新服务器到监控列表，以便集中管理
- 作为开发者，我希望能够为不同环境的服务器打标签，以便快速筛选

**验收标准**
- ✅ 支持通过 UI 表单添加服务器
- ✅ 支持配置服务器名称、地址、端口、适配器类型
- ✅ 支持启用/禁用服务器监控
- ✅ 支持删除服务器及其历史数据

---

### 3.2 主机指标监控

**功能描述**
监控服务器核心资源使用情况：
- **CPU**: 总体使用率、单核使用率、负载均衡
- **内存**: 总量、已用、可用、使用率
- **磁盘**: 分区空间、I/O 读写速率
- **网络**: 接口流量、发送/接收速率

**用户故事**
- 作为运维工程师，我希望看到 CPU 使用率的实时曲线，以便发现性能峰值
- 作为开发者，我希望看到内存使用趋势，以便优化应用内存占用

**验收标准**
- ✅ 实时显示 CPU/内存/磁盘/网络指标
- ✅ 支持时间范围选择（1h/6h/24h/7d）
- ✅ 支持多服务器对比视图
- ✅ 支持导出历史数据（CSV/JSON）

---

### 3.3 网络质量探测

**功能描述**
主动探测服务器网络质量：
- **Ping**: ICMP 延迟、丢包率
- **TCP**: 端口连通性、连接延迟
- **DNS**: 域名解析时间

**用户故事**
- 作为运维工程师，我希望监控服务器之间的网络延迟，以便发现网络故障
- 作为开发者，我希望测试 API 端点的 TCP 连通性，以便排查连接问题

**验收标准**
- ✅ 支持从桌面端发起探测（desktop-local）
- ✅ 支持从远程代理发起探测（remote-agent）
- ✅ 支持自定义探测目标和频率
- ✅ 显示探测结果时序图

---

### 3.4 流量分析

**功能描述**
- 网络接口流量排行
- 流量突发检测
- 带宽使用趋势分析

**用户故事**
- 作为运维工程师，我希望看到哪个网络接口流量最大，以便优化网络配置
- 作为开发者，我希望检测流量突发，以便发现异常流量

**验收标准**
- ✅ 显示接口流量 Top N 排行
- ✅ 标记流量突发事件
- ✅ 支持按时间段聚合流量统计

---

### 3.5 告警系统

**功能描述**
- 基于阈值的告警规则
- 持续时间判定（避免瞬时抖动）
- 冷却时间机制（防止告警风暴）
- 桌面通知推送

**用户故事**
- 作为运维工程师，我希望 CPU 使用率超过 80% 持续 5 分钟时收到告警，以便及时处理
- 作为开发者，我希望磁盘空间不足 10% 时收到通知，以便清理日志

**验收标准**
- ✅ 支持创建/编辑/删除告警规则
- ✅ 支持条件：大于/小于/等于
- ✅ 支持持续时间和冷却时间配置
- ✅ 支持桌面通知（macOS/Windows/Linux）
- ✅ 显示告警历史记录

---

### 3.6 数据存储与保留

**功能描述**
- 三层数据保留策略：
  - 原始数据：7 天，全分辨率
  - 1 分钟聚合：30 天
  - 15 分钟聚合：90 天
- SQLite 本地存储，WAL 模式

**用户故事**
- 作为运维工程师，我希望查看过去 7 天的详细数据，以便精确分析问题
- 作为开发者，我希望查看过去 3 个月的趋势，以便容量规划

**验收标准**
- ✅ 自动清理过期数据
- ✅ 支持手动触发数据清理
- ✅ 显示数据库大小和占用空间

---

## 4. 非功能需求

### 4.1 性能要求
- 单服务器轮询间隔：5-60 秒可配置
- 支持同时监控 50+ 服务器
- UI 响应时间 < 100ms
- 数据库查询时间 < 500ms

### 4.2 可用性要求
- 桌面应用启动时间 < 3 秒
- 网络故障时自动重试，指数退避
- 数据库损坏时自动恢复（WAL 模式）

### 4.3 安全要求
- Go Agent 使用 Bearer Token 认证
- 支持 SSH 隧道访问私有网络服务器
- 本地数据库加密（可选）
- 不存储明文密码

### 4.4 兼容性要求
- 操作系统：macOS 11+、Windows 10+、Linux (Ubuntu 20.04+)
- 数据源：node_exporter 1.0+、Glances 3.0+、自定义 Go Agent
- 浏览器内核：Tauri WebView (Chromium-based)

### 4.5 可维护性要求
- 代码覆盖率 > 70%
- 关键路径单元测试覆盖率 > 90%
- 遵循 Rust/TypeScript/Go 最佳实践
- 完整的 API 文档和架构文档

## 5. 技术架构

### 5.1 技术栈
- **前端**: React 18 + TypeScript + Vite + Tailwind CSS + Zustand
- **后端**: Rust + Tauri 2 + SQLite + tokio
- **远程代理**: Go + Gin + gopsutil
- **图表**: Recharts
- **通知**: tauri-plugin-notification

### 5.2 架构模式
- **适配器模式**: 支持多种数据源（node_exporter、Glances、Go Agent）
- **三域分离**: 主机指标、网络质量、流量分析独立模块
- **分层存储**: 原始数据 → 1 分钟聚合 → 15 分钟聚合

## 8. 风险与挑战

### 8.1 技术风险
- **SQLite 性能瓶颈**: 大量服务器时写入压力大
  - 缓解措施：批量写入、WAL 模式、定期清理
- **网络探测权限**: ICMP Ping 需要 root 权限
  - 缓解措施：提供 TCP Ping 替代方案

### 8.2 产品风险
- **用户学习成本**: 配置复杂度高
  - 缓解措施：提供配置向导、预设模板
- **竞品压力**: Prometheus + Grafana 生态成熟
  - 差异化：轻量级、开箱即用、桌面原生

## 9. 附录

### 9.1 术语表
- **Adapter**: 数据源适配器，用于从不同监控系统获取指标
- **Vantage Point**: 观测点，标识指标来源（桌面端/远程代理）
- **Counter**: 累计计数器，单调递增的指标类型
- **Gauge**: 瞬时值，可增可减的指标类型
- **Rollup**: 数据聚合，将高分辨率数据降采样
